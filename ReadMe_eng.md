# 📊 Discord Server Activity (SCP:CS) :: ALPHA 0.1V
<img alt="Static Badge" src="https://img.shields.io/badge/By%20Smoxeq%20-%20scp%3Acs?style=for-the-badge&logo=scpfoundation&logoColor=black&label=SCP%3ACLASSIFIED%20SITE%20API&color=blue">

This plugin allows you to display dynamic statistics of your server in a Discord channel. Instead of spamming new messages, the plugin **edits** a single existing message, updating the player count in real-time.

## ⚙️ Available Configurations (Config)

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `Servername_activity` | string | Your server's name, which will appear in the Embed title. |
| `URLHook_activity` | string | Your Discord Webhook URL. |
| `maxplayers` | int | Maximum number of players (for display limits). |
| `DiscordMessageId` | string | The ID of the message to be edited (see instructions). |

---

## 🚀 Installation & Setup Guide

To ensure the plugin works in "update" mode rather than "spam" mode, follow these steps:

1. **Silent Mode:** It is recommended to perform the setup when there are **no players** on the server.
2. **Initial Launch:** * Fill in only `URLHook_activity` and `Servername_activity`.
   * Leave the `DiscordMessageId` field **empty**.
   * Start the server.
3. **Getting the ID:** * Once the round starts, a new message (Embed) will appear in Discord.
   * Right-click the message and select **"Copy Message ID"** (Developer Mode must be enabled in Discord).
4. **Finalizing:** * Paste the copied ID into the `DiscordMessageId` config field.
   * Restart the server (wait for the round to end). Now the plugin will edit this specific message every time the server or round restarts.

> [!WARNING]
> If you leave the `DiscordMessageId` field empty, the plugin will send a **new message at every round start** without deleting the old ones.

---

## 🛑 What NOT to do
* **Do not delete the message** in Discord after saving its ID in the config. If the message is deleted, the plugin won't be able to edit it and will trigger an error.
* **Do not enter a random ID** manually. This will cause delivery errors. The ID must be generated by the webhook itself during the first launch when `DiscordMessageId` is empty.
* **Avoid frequent server restarts** during peak player counts to avoid hitting Discord API Rate Limits.
